Privacy Policy — Medusa Helper

Last updated: July 25, 2026

This Privacy Policy explains what data Medusa Helper ("the Bot", "we", "us") collects, why, and how it is used when you or your Discord server use the Bot's features. By using the Bot, you agree to the collection and use of information as described in this policy.

1. Information We Collect

1.1 Information You Provide Directly

When you use the Bot's slash commands, context menu actions, or administrator setup features, we may process:

  • Discord data: your Discord user ID, username, server ID, and the slash command or context menu inputs needed to process your request (for example /qris, /calc, /check, /order, /payment, /leaderboard, Apps > Giveaway Check, and Apps > Upload Payment).
  • Target message data: when a server staff member uses a message context menu such as Apps > Giveaway Check or Apps > Upload Payment, we process the selected message's visible fields, embeds, attachments, and image URLs only as needed to complete that action.
  • Roblox data: Roblox usernames and user IDs you submit or that are submitted on your behalf (for example via /check, /giveaway, or /order), used to verify group membership and process Robux orders.
  • Payment-related data: order amounts, order numbers, payment proof images (screenshots) submitted via /payment or Apps > Upload Payment, and order metadata (order number, status, total price) returned by our backend.
  • Server configuration data: if you are a server administrator, data you configure via slash commands, such as QRIS merchant payment details (/qrissetup), leaderboard channel settings (/leaderboardset), and role permissions (/setrole).

1.2 Information Collected Automatically

  • Command usage logs: for debugging and reliability purposes, we log which slash commands or context menu actions were invoked, by whom (Discord user ID), in which server, and basic technical details (e.g. HTTP response statuses from external APIs).

1.3 Information We Do Not Collect

  • We do not read ordinary conversation messages for prefix-based command detection.
  • We do not require Discord Message Content Intent for normal operation.
  • We only process a message's content or attachments when a user explicitly targets that message through a message context menu action.
  • We do not use Discord's Server Members list or presence/status data.
  • We do not use any collected data to train machine learning or AI models.

2. How We Use Information

We use the information described above to:

  • Execute the action you requested (e.g. generate a QRIS payment QR code, calculate Robux conversion rates, check Roblox group membership eligibility, verify giveaway participation, place Robux orders, upload payment proof, or render the leaderboard).
  • Verify eligibility for group-fund orders and giveaways by checking Roblox group membership via the Roblox API.
  • Communicate with our own backend order-processing system (medusablox.com) to place orders and confirm payments on your behalf.
  • Maintain and improve the reliability of the Bot (via debug logs).
  • Enforce server-specific configuration set by administrators (e.g. which roles can use /order, /payment, or Apps > Upload Payment).

3. How We Store and Share Information

  • Server configuration (QRIS merchant details, leaderboard channel, allowed roles) is stored in a configuration file on our hosting server, associated with your Discord server ID. This is not shared with third parties.
  • Order and payment data is transmitted to our own backend service (medusablox.com) to process Robux orders and payment confirmations. This data is not sold or shared with unrelated third parties.
  • Roblox lookups (username/group membership checks) are sent to Roblox's public APIs (users.roblox.com, apis.roblox.com) solely to verify eligibility; we do not control Roblox's own data practices for these requests.
  • We do not sell your data to advertisers or unrelated third parties.

4. Data Retention

  • Server configuration data is retained for as long as your server uses the Bot, or until an administrator removes it (e.g. via /qrisreset, /leaderboardset remove, /setrole clear).
  • Order, payment, and eligibility-check data is retained only as long as needed to complete and support the relevant transaction, and for basic operational logging.
  • You may request deletion of your data by contacting us (see Section 7).

5. Your Choices

  • Command-based data: You can avoid most data processing simply by not invoking the Bot's slash commands or message context menu actions.
  • Server admins can remove stored configuration at any time using /qrisreset, /leaderboardset remove, and /setrole clear.
  • You may contact us to request access to, correction of, or deletion of your data (see Section 7).

6. Children's Privacy

The Bot is not directed at children under 13 (or the minimum age required by Discord's Terms of Service in your region). We do not knowingly collect personal data from children.
